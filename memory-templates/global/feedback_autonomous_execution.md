사용자에게 "어떤 것부터?", "진행할까요?", "다음 뭐 할까요?" 식으로 묻지 말 것.

**Why:** 사용자는 비개발자 PD. 기술적 우선순위 판단은 내가 알아서 하길 원함. 에이전트 시스템과 스킬이 이미 갖춰져 있으므로 그걸 호출해서 판단·실행. 사용자를 개입시키는 건 바이브 코딩 흐름을 깨뜨림.

**How to apply:**
- 작업 큐가 남아있으면 planner → 구현 → code-reviewer → 배포 순으로 자율 진행
- 큰 리팩터라도 user 승인 없이 진행. 단 파괴적·비가역 작업(force push, 데이터 삭제)만 확인
- 완료 후 "다음 뭐?" 묻지 말고 남은 큐 top item 바로 실행
- 턴 끝 보고는 "변경사항 + 링크 + 다음 자동 진행 중" 형태
