프로젝트 작업(추천·분석·코드 수정·더미 주입·에이전트 브리핑 등)을 시작하기 전에 `memory/projects/project_<project>_*.md` 중 작업 영역과 매칭되는 파일을 먼저 Read한다. 키워드 매칭 대상: 광고, 동기화, API 키, 파싱, 스키마, 배포, 진행 현황 등 도메인 키워드. 매칭 파일이 하나라도 있으면 선독하지 않은 상태로 작업 시작 금지.

**Why:** 2026-04-18 [예시 프로젝트 A] 작업에서 `project_[예시 프로젝트 A]_ad_logic.md` (광고 SSOT, `rp_visited` 게이팅 명시)를 읽지 않은 채 스크롤 수정 + 더미 주입을 진행해 "광고 영역 사라짐 / 카드 간격 붕괴" 증상을 사용자가 직접 지적해서 발견. 해당 메모리 파일은 심지어 "사용자가 지적해서 추가"라고 경위까지 명시돼 있었음. 같은 부류 실수의 반복.

**How to apply:**
- L1이 사용자 요청을 받으면 → 작업 프로젝트 식별 → 해당 프로젝트의 projects/ 메모리 목록 확인 → 작업 영역 키워드와 매칭되는 파일 Read → 그 다음 코드/에이전트 단계
- L2/L3 에이전트 브리핑에도 관련 메모리 요약을 포함해 전달
- 메모리 상단의 `<system-reminder>`에 나오는 "point-in-time observation, verify against current code" 경고는 참조 우선순위를 낮추는 이유가 아니라 **검증 절차를 추가하는** 이유. 메모리 → 최신 코드 대조 순.
- 대상 프로젝트의 projects/ 메모리가 없으면 이 룰은 스킵 가능하지만, 있는데 안 본 경우는 작업 중단하고 먼저 읽기.
