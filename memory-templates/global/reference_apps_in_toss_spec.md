# 앱인토스 기술 스펙 (2026-04 검증)

토스 미니앱 설계·기획 시 이 파일 먼저 확인. 세부는 공식 문서 URL 따라가 재검증. 토스 정책은 변할 수 있으니 6개월 이상 경과 후 핵심 항목(특히 푸시·심사기간·수수료)은 최신 확인 필수.

## ⛔ 절대 불가능한 것들 (중요)

| 기능 | 상태 | 근거 |
|---|---|---|
| **로컬 예약 알림** (앱 꺼진 상태서 OS 차원 정확 시각 알람) | ❌ 미지원 | [공식 답변 Dylan 2026-04-14](https://techchat-apps-in-toss.toss.im/t/topic/3412) |
| **알람앱급 우선순위·전면 표시·커스텀 알림음** | ❌ 미지원 | 동일 |
| **다크모드** | ❌ 라이트 고정 | FAQ |
| **다크패턴 UI** (갑툭튀 바텀시트, 숨긴 종료, 기만 CTA) | ❌ 심사 탈락 | FAQ |

**핵심 함의**: "사용자가 잊어도 정확한 시점에 앱이 깨워주는" 형태의 리마인더·알람·캘린더 미니앱은 플랫폼 근본 궁합이 안 맞음. 구현하려면 스마트 메시지(기능성) + 내 서버 cron 조합 필요하며 난이도 "중상".

## ✅ 제공되는 기능 (SDK)

카메라, 연락처, 위치, 저장소(ephemeral), 햅틱, 클립보드, 공유, 인앱결제, 광고(전면·리워드), 네트워크 상태, OS/로케일, 토스 로그인·결제·인증, 딥링크.

예제 저장소: [toss/apps-in-toss-examples](https://github.com/toss/apps-in-toss-examples) (TS 77%). Cocos Creator용 별도 레포 있음.

## 📩 푸시 알림 — 조건부 가능 (스마트 메시지)

**두 종류**:
- **푸시**: 앱 닫힌 상태에서 OS 레벨로 뜨는 메시지 (토스 앱 이름·로고로 표시)
- **알림**: 토스 우상단 종 아이콘 앱 내 메시지. 주목도 낮음

**발송 방법**: 스마트 메시지(기능성/광고성) API를 파트너사 서버에서 호출. 즉시 발송 또는 세그먼트 기반 트리거.

**기능성 메시지 조건**:
- 광고 내용 금지. "서비스 이용에 직접 필요한 정보"만
- 템플릿 **사전 검수 필수** (영업일 2~3일)
- 템플릿 자체는 재사용 가능. 동적 파라미터 `{{ variable }}` 지원 ([근거](https://techchat-apps-in-toss.toss.im/t/url/3297))
- 사용자에게 "서비스 이용 수신 동의" 필요

**광고성 메시지 조건**:
- "마케팅 수신 동의"한 유저 대상
- 토스 자동 최적화 작동 (발송 시각·세그먼트 최적화). 개발자 정밀 통제 어려움

**"특정 유저 × 특정 시각" 발송**:
- 파트너 서버에서 cron 돌려 조건 맞는 유저 추려 API 호출 → 해당 유저에게 푸시
- 세그먼트 기반 타깃이 주력. 1명 단위 API 발송은 가능하지만 별도 확인 필요

## 💾 저장소 — 백엔드 필수

| 옵션 | 보존 |
|---|---|
| WebView 로컬 저장소 | 앱 삭제 시까지만 |
| 영구 데이터 | **파트너사 백엔드 서버 필수** |

**영향**: 유저 데이터·히스토리 필요한 미니앱(대부분의 유틸·커뮤니티·커머스)은 서버 구축 필수. 비개발자 바이브 코딩이라면 Supabase·Vercel 같은 BaaS 조합 추천.

## 🔐 서버 연동 요구사항

- **mTLS 상호 인증 필수** (server-to-server API 호출 시)
- **CORS 화이트리스트**: `https://<appName>.apps.tossmini.com` 도메인 허용
- 테스트 환경과 프로덕션 환경의 CORS 정책·네트워크 동작이 다름 → 프로덕션 별도 검증 필수

## 📦 배포·심사

| 항목 | 값 |
|---|---|
| 번들 크기 | 압축 해제 100MB 이하 |
| 최소 OS | Android 7+, iOS 16+ |
| 최소 연령 | 만 19세+ |
| 계정 심사 | 1-2영업일 |
| **출시 심사 (게임)** | **2-4주** (공식) |
| **출시 심사 (비게임)** | **공식 2-3개월 / 사용자 실측 약 1주** (2026-04 개인 경험치. 카테고리·복잡도에 따라 변동 가능) |
| 업데이트 심사 | 영업일 3일 |
| 푸시 템플릿 검수 | 영업일 2-3일 |

**함의**: 유틸리티·커머스·도구류는 첫 출시까지 최소 2~3개월. 게임은 상대적으로 빠름.

## 💰 수익 구조

| 항목 | 토스 수수료 |
|---|---|
| 광고 매출 (전면·리워드) | **15%** |
| 인앱 결제 | **5%** |
| 토스페이먼츠 별도 승인 | 7-14영업일 |

**인앱 결제 단가 범위**: 400원 ~ 1,400,000원

**광고 유형**: 전면(interstitial) + 리워드 광고(rewarded)만. 홈배너·네이티브는 플랫폼 차원에서 통합 운영.

## 🧭 기획 판단 가이드 (이 스펙 기반)

1. **"리마인더형" 미니앱**은 플랫폼 궁합 나쁨 (로컬 예약 불가). 꼭 하려면 서버+mTLS+cron 각오.
2. **유틸 1st 출시 기준 리드타임** = 코드 1~2주 + 심사 2~3개월. 기회비용 고려 후 아이템 선정.
3. **빠른 출시 원하면 게임** (심사 2~4주). 단 레드오션.
4. **수익 최대화** = 광고(15% 수수료) + 외부 제휴(CPA). 인앱결제(5%)는 범위 400원~140만원.
5. **백엔드 각오**: 영속 데이터 필요하면 서버 필수. Supabase·Vercel 조합 추천.
6. **bundle 100MB** 한도. 이미지·영상은 CDN 외부화.
