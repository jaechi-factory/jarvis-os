# 제품 기획 — "Fire and Forget" 기본 원칙

앱인토스 미니앱, 바이브코딩 프로덕트 기획 시 기본 필터:

**출시 후 사용자의 직접 운영 공수가 최소여야 함.** 주 단위 콘텐츠 큐레이션·딜 업데이트·수동 큐레이션 같은 운영-heavy 구조는 사용자가 명시적으로 수락하기 전까지 상위 후보에서 배제.

**Why**: 2026-04-18 펫딜(반려동물 쿠폰 큐레이터) 추천 과정에서 "주 2회 × 1.5hr = 주 3hr 운영" 가정을 당연시했다가 사용자가 지적 — "내가 이걸 만들고 나서, 직접적 공수가 안 드는 방향이어야 해." 사용자는 한국 프로덕트 디자이너 1인 비개발자로서 처음부터 "만들어놓고 알아서 돌아가는" passive income 구조를 기대하고 있었음.

**How to apply**:
- 후보 매트릭스 초기 단계부터 "출시 후 주간 운영 시간 추정" 컬럼을 필수로 포함
- **주 1시간 초과 = 감점**, 주 3시간 초과 = 기본 배제 (사용자 명시 수락 시에만 진입)
- 자동화 친화 구조 우선: AI 자동생성(GPT/Gemini), 정적 계산 도구, 로직 고정 유틸, 일회성 사용 앱
- 후보에 운영 부담이 있으면 반드시 "이 아이템은 주 N시간 운영이 필요한데 괜찮아?" 선확인
- 참고: "콘텐츠를 매일·매주 업데이트해야 살아있는" 구조(쿠폰 큐레이션·핫딜·뉴스 큐레이션·딜 콜렉션)는 passive 아님

**예외 조건**: 사용자가 "운영 할 수 있어", "콘텐츠 매일 올릴게" 같이 명시 수락 시에만 운영-heavy 아이템을 상위 후보에 올림.
