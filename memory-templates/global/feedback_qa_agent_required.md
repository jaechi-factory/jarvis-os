type: feedback

내가 직접 구현한 경우에도 배포 전 qa 에이전트를 호출해야 한다.

**Why:** QA 생략 → 빌드 에러로 배포 실패한 사례 발생. qa 에이전트가 `npm run build` 검증과 회귀 케이스 확인을 담당하는데, 이를 건너뛰면 Vercel에서 에러가 나서야 문제를 발견하게 됨.

**How to apply:**
- dev 구현(또는 내가 직접 구현) → qa 호출 → qa 통과 후 push
- qa에게 넘길 때 반드시 포함할 것: 수정한 파일 목록, 변경 내용 요약, `npm run build` 실행 요청
- qa 없이 push하는 유일한 예외: 오타 수정, 주석 변경 등 코드 로직 무관한 변경
