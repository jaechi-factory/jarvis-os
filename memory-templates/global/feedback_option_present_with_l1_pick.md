사용자에게 선택지를 제시할 때 L1(나)의 추천 안과 근거를 항상 같이 표기한다. "중립 제시만" 금지.

**Why**: 2026-04-19 하루말씀 Library AppBar 중복 건 — 사용자가 "옵션 나열 뒤에 너 생각도 항상 같이 알려줘, 판단에 참고하게"라고 명시. 옵션만 던지면 비개발자 사용자 입장에서 결정 부담이 큼. L1의 의견이 있어야 "동의/반박" 기준점이 생김.

**How to apply**:
- A/B/C 옵션 비교 표/리스트 제시할 때 **마지막 줄에** "**L1 추천: X — 근거: Y**" 반드시 포함
- 표의 한 축으로 "L1 의견" 컬럼을 두거나, 표 바로 아래 한 단락으로 추천 명시
- 추천 없이 옵션만 나열 = 규칙 위반. "사용자가 결정하세요" 만으로 끝내지 않기
- 이유는 1~3줄로 구체적으로. 모호한 "둘 다 좋음" 금지
- 단, **진짜로 동급 트레이드오프**일 땐 "동급 — 선호도 문제"라고 명시하고 각 안의 적합 시나리오 구분 제시
- Confidence 표시 선택 가능 (high/med/low)
- 재질문 받으면(예: "뭐가 더 나아?") 더 단호한 의견 + 근거 보강. 회피 금지
