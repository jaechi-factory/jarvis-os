**이 규칙은 [예시 프로젝트 A]뿐 아니라 모든 프로젝트·모든 세션에 무조건 적용된다.** 모든 다단계 작업(**4단계 이상**, 또는 여러 도메인 걸치는 모든 작업)은 시작 전에 체크리스트를 먼저 출력하고, 진행되는 항목을 실시간으로 [x] 체크하여 사용자에게 보여준다.

**2026-04-18 임계값 조정:** 3단계 → 4단계. 단순 3단계 작업(원복→수정→빌드 등)까지 체크리스트 돌리는 건 토큰 낭비. 4단계 이상부터 적용.

**Why:** 2026-04-18 [예시 프로젝트 A] 작업 중 QA 호출 스킵 + close-out(메모리/브랜치/빌드/배포) 자율 실행을 시도함. MD에 "code-reviewer 필수" 명시돼 있었는데도 스킵했고, 사용자가 "과정 마음대로 스킵하래? 그럶거면 md는 뭐하러 쓰고 기억하냐"고 강하게 지적. 체크리스트가 없으면 내부 플로우가 사용자에게 블랙박스라 스킵 여부 감시 불가.

**How to apply:**
- 작업 시작 즉시 전체 단계를 `- [ ] 1. ...` 형태의 체크리스트로 메시지 본문 앞부분에 출력
- 각 단계 완료할 때마다 다음 메시지에 체크리스트 전체 다시 출력하며 완료 항목을 `[x]`로 표시 (상태 가시화)
- 임의 순서 변경·스킵 금지. 순서 바꿔야 하면 사유 말하고 사용자 승인 먼저
- 특히 `feedback_qa_agent_required.md`, `feedback_build_validation.md`, `feedback_deploy_direct.md`, `feedback_project_memory_first.md` 같은 **선행 규칙 메모리는 체크리스트에도 명시 항목으로 박기**
- "자율 실행" 명분으로 체크리스트 단계 건너뛰기 금지. 자율 실행은 체크리스트 **위**에서 돌아가야 함
- 단순 1~3단계 작업(단일 파일 편집, 조회, 원복→수정→빌드 같은 짧은 체인)은 체크리스트 불필요
