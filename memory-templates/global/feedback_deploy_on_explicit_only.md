모든 프로젝트·모든 세션에서 아래 동작은 **사용자 명시 지시 있을 때만** 실행한다. 체크리스트에 올려도 "의사결정 대기" 상태로 두고, 자동 진행 금지.

**적용 대상 동작:**
- `git commit` (feat/fix 브랜치 포함)
- `git merge` (main 머지)
- `git push` (모든 브랜치)
- Vercel 배포 (`vercel --prod`, 자동배포 대기 포함)
- `.ait` 파일 생성 (`npx ait build`)
- 바탕화면 `.ait` 복사

**Why:** 2026-04-18 [예시 프로젝트 A] 작업 중 문제 해결 턴마다 체크리스트에 커밋·머지·배포·AIT를 자동 단계로 넣었더니 사용자가 "문제 해결중인데 할때마다 하면 어떡해"라고 지적. 문제 해결은 여러 번 반복되는데 매번 배포 파이프라인 돌리면 히스토리 오염되고 불필요한 배포 발생.

**How to apply:**
- 체크리스트에 이 단계들은 넣지 말거나, 넣더라도 "— (사용자 명시 지시 시에만)" 표기로 자동 실행 안 함을 명시
- 여러 문제 의사결정·구현 끝나고 **사용자가 "이제 커밋해" / "배포까지 해" / "AIT 만들어"** 등 명시적으로 말할 때 한꺼번에 실행
- 구현·수정·빌드 검증·QA 리뷰는 자동 진행 허용. 원격·공유 상태 변경만 명시 지시 대상
- 사용자가 "전체 완료해줘" 등 포괄적으로 승인하면 그때 배포 파이프라인 가동
- 이전 룰인 `feedback_deploy_direct.md` (Vercel 자동배포 의존 금지, vercel --prod 직접 실행)은 유효. 다만 **실행 시점**은 사용자 명시 지시 후
